# AI坊学生管理系统 - 项目总览

## 项目概述
AI坊学生管理系统是厦门工学院人工智能创作坊的综合学生管理解决方案，包含前端和后端两个核心部分，提供学生信息管理、分时段签到、数据可视化等完整功能。

## 项目结构

### 📁 前端
- **开发记录**：2025年9月、10月、12月的前端开发历程
- **技术栈**：Vue3 + Element Plus + ECharts + Pinia + Vue Router
- **核心功能**：用户认证、分时段签到、数据可视化、响应式设计、主题切换
- **详细信息**：查看 [前端README](前端/README.md)

### 📁 后端
- **开发记录**：2025年9月、10月、12月和2026年3月的后端开发历程
- **技术栈**：Spring Boot 3.5.5 + Spring Security + JWT + MySQL + Redis
- **核心功能**：学生信息管理、分时段签到系统、安全认证、数据统计、积分管理
- **详细信息**：查看 [后端README](后端/README.md)

## 技术架构

### 前端技术栈
- **框架**：Vue 3.x（Composition API模式）
- **UI组件库**：Element Plus
- **状态管理**：Pinia
- **路由管理**：Vue Router
- **数据可视化**：ECharts
- **HTTP客户端**：Axios
- **开发工具**：TypeScript、ESLint、Vue CLI

### 后端技术栈
- **框架**：Spring Boot 3.5.5（Java 24）
- **数据访问**：Spring Data JPA + Hibernate
- **安全认证**：Spring Security + JWT
- **数据存储**：MySQL 8.0 + Redis
- **缓存优化**：Redis缓存签到状态，响应时间优化至100ms内
- **文件存储**：MinIO（学生打卡照片存储）
- **接口文档**：Knife4j 4.6.0

## 核心功能模块

### 1. 学生信息管理
- **信息采集**：姓名、性别、电话、年级、班级等信息
- **CRUD操作**：学生注册、登录、信息修改、删除
- **数据验证**：学号格式、手机号唯一性、必填字段等业务规则校验

### 2. 分时段签到系统
- **时段控制**：早（8:00-11:00）、午（14:00-17:00）、晚（19:00-22:00）
- **防重复机制**：基于Redis缓存实现毫秒级重复签到判断
- **安全验证**：IP白名单限制 + 验证码机制，杜绝刷签到
- **拍照验证**：前端通过Browser Camera API调用摄像头，拍摄完成后上传至MinIO

### 3. 数据可视化
- **大屏展示**：打卡排名、今日/日均打卡、年级/专业分布
- **图表类型**：柱状图、环形饼图、词云图
- **实时数据**：15秒自动刷新，底部进度条显示刷新状态
- **多维度统计**：学生分布、考勤数据、积分排名等

### 4. 安全认证
- **JWT Token**：7天有效期，支持Token生成、验证、解析
- **IP白名单**：验证码获取接口仅允许白名单IP访问
- **权限控制**：基于学生等级的权限管理系统

### 5. 积分管理
- **积分记录**：签到次数 × 10的积分规则
- **排名系统**：积分排名查询，支持前N名学生查询
- **改分记录**：管理员可调整积分，记录改分理由

### 6. 头像管理
- **上传功能**：支持多种图片格式，自动压缩至200KB以下
- **存储管理**：上传新头像时自动清理旧头像文件
- **获取接口**：根据学生ID获取头像图片
